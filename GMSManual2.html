Game Management System Guide

The intended audience for this guide is lottery users who only access the Game Management System’s (GMS) Retailer and Owner modules. GMS is the master control center for administrating and managing all draw games, instant games, accounting, reports, terminals, retailers, and store owners. This guide is used by SG's training team during lottery staff training.

    Documentation Process

    1. If necessary, meet with the project manager to obtain the names and contact information of each team member. (software engineers, SMEs, product manager, trainers, lottery staff, etc.)

    2. Work together with SG’s training team and our customer, the lottery, to gather data regarding the target audience and their needs. We usually create a user authorization matrix spreadsheet that specifies user access.

    3. Compile all software-related documents (system requirements, functional specs, addendums, etc.) from our department’s repository. These documents were usually written by me or another senior technical writer on the team.

    4. Work together with the product manager, SMEs, software engineers, and software architect to obtain access to the system and acquire any additional information I may need about it.

    5. Once I get access to the system, I start using the software and writing down any questions I have about it.

    6. If necessary, request a 30-minute to an hour meeting with the software engineers to address any questions or concerns I have.

    7. Create an outline based on the information I acquired.

    8. If necessary, go over the outline with the lottery, SG’s training team, and other important stakeholders.

    9. Once we reach a consensus, I would write the initial draft.
